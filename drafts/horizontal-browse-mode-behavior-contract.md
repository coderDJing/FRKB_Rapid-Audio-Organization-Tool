# Horizontal Browse 双轨/编辑模式用户操作手册草案

## 1. 文档用途

本文只定义用户在 Horizontal Browse 双轨模式和编辑模式里操作时，界面、声音和状态应该发生什么。

本文不规定技术实现方式，不描述代码结构，不写时间公式，不教开发者怎么写代码。后续重写时，以本文作为产品表现和验收标准。

基础原则：

- 除“Beat Sync 开启时点击左右移动网格线按钮”的新需求外，原来已经满意的操作表现必须保持。
- 用户不需要学习新的保存流程，不需要点击“保存为新的网格”，也不需要确认“应用相位”。
- 每个操作都必须让界面、声音、按钮状态保持一致。
- 不允许出现用户看见 A、听见 B、按钮状态又显示 C 的分裂表现。
- 不允许用卡顿、爆音、明显跳动换取表面正确。

## 2. 当前遇到的问题

### 2.1 最初需求

用户最初要解决的问题只发生在双轨模式 Beat Sync 开启时。

前提：

- 两轨已经进入 Beat Sync。
- 用户点击某轨的左移大步、左移小步、右移小步、右移大步网格线按钮。

用户期望：

- 当前画面里的视觉网格保持对齐。
- 两轨视觉网格仍然对齐。
- 波形移动。
- 声音跟随波形移动。
- Beat Sync 不取消。
- 不弹出新的保存流程。
- 不改变用户原来的操作风格。

也就是说，用户仍然像以前一样点四个左右移动按钮；只是当 Beat Sync 开启时，按钮仍然保存真实网格，但当前双轨画面里的可感知结果不应该是两轨网格漂开，而应该是网格继续保持对齐，歌曲内容相对网格移动。

### 2.2 已经复现过的错误表现

前面几轮改动后，出现过这些错误表现：

- 点击四个左右移动网格线按钮后，Beat Sync 被变成取消状态。
- 后来 Beat Sync 不再被取消，但两轨网格线变得不对齐。
- 日志加入后，问题仍然反复出现，没有形成稳定正确的行为。
- 看起来某一层修对了，另一层又出现声音、波形、网格或 sync 状态不一致。
- 用户多次复测后，问题依旧，没有达到“真实网格保存、当前视觉网格对齐、波形动、声音跟着动、sync 不变”的目标。

这些错误表现说明，继续在现有改动上打补丁不可靠。

### 2.3 当前不能接受的结果

以下结果都不能作为修复完成：

- Beat Sync 不取消了，但网格线不对齐。
- 网格线不动了，但声音没跟着波形动。
- 声音跟着动了，但波形显示不对。
- 某一次点击看起来对，连续点击后累计偏移不对。
- 暂停时看起来对，播放后声音不对。
- 播放时听起来对，但按钮状态或 sync 状态不对。
- 为了避免复杂性，要求用户额外点击“保存为新的网格”。
- 为了让当前补丁跑通，破坏原来已经满意的双轨和编辑模式其他行为。

## 3. 初步调研发现的深层问题

### 3.1 缺少一份完整的交互标准

这次问题暴露出一个更大的问题：双轨模式和编辑模式的功能很多，但之前缺少一份完整、无歧义的用户操作标准。

结果就是：

- 修一个按钮时，容易误伤播放。
- 修 sync 时，容易误伤网格。
- 修波形时，容易误伤声音。
- 修双轨时，容易误伤编辑模式。
- 修当前问题时，容易破坏用户原来已经满意的操作表现。

因此，重构前必须先用本文把用户操作表现写清楚。

### 3.2 “网格移动”和“歌曲移动”在用户体验上是两件事

用户点击同一组左右按钮，在不同状态下实际想要的结果不同。

Beat Sync 关闭时：

- 用户是在编辑歌曲真实网格。
- 所以网格线应该移动。
- 这个修改应该保存为真实网格。

Beat Sync 开启时：

- 用户不是要破坏两轨视觉对齐。
- 用户不是要弹出一套新的保存流程；真实网格修正仍然应该按原有方式保存。
- 用户是想在保持同步网格对齐的前提下，微调歌曲内容相对网格的位置。
- 所以当前画面里的视觉网格线应该保持对齐，波形和声音应该移动。

如果系统把这两种体验混成一件事，就会出现现在这种问题：要么 sync 掉，要么网格错，要么声音和波形分裂。

### 3.3 “看见的网格”“看见的波形”“听见的声音”必须同时定义

这次问题不是单纯按钮方向错，也不是单纯保存字段错。

真正需要同时满足的是三件事：

- 用户看见的网格线在哪里。
- 用户看见的波形内容在哪里。
- 用户听见的声音从哪里来。

只修其中一个都不够。

错误例子：

- 只让网格不动，但声音仍按旧位置播放，用户会听见错位。
- 只让声音移动，但波形不动，用户会看见错位。
- 只让波形移动，但 sync 状态掉了，用户会丢失双轨同步。
- 只让 sync 按钮保持亮，但两轨视觉网格错开，用户仍然认为同步坏了。

### 3.4 Beat Sync 状态不能被普通微调操作误伤

Beat Sync 是用户明确开启的状态。

点击左右移动网格线按钮时，如果用户没有主动关闭 sync，系统不应该把 sync 变成关闭。

如果某个操作确实会导致完整同步暂时无法维持，也必须表现成明确、可理解的提示态，而不是静默取消或状态乱跳。

这次出现“点击四个按钮后 Beat Sync 取消”，说明按钮操作和 sync 状态之间的边界不清楚。

### 3.5 原有满意行为必须作为重构基线

用户已经明确说明：在这次所有改动之前，除了最初提出的 Beat Sync 下左右移动网格线问题，其他功能表现都是满意的。

这句话是重构边界：

- 不要借重构机会改掉原来的操作风格。
- 不要因为一个 sync 问题重写出一套陌生交互。
- 不要把原来能用的 cue、loop、seek、drag、编辑模式搞坏。
- 不要把临时调试逻辑当成新产品规则。

重构目标不是“做一个全新的双轨模式”，而是“保留原来满意的双轨/编辑体验，只把这次暴露出的网格、波形、声音、sync 边界理顺”。

## 4. 通用界面对象

### 4.1 轨道

Horizontal Browse 里有两条轨道：

- 上轨：top deck。
- 下轨：bottom deck。

双轨模式显示上轨和下轨。编辑模式只显示上轨。

### 4.2 波形

每条可见轨道有两种波形：

- 概览波形：显示整首歌，用于快速跳转和查看整体结构。
- 大波形：显示当前播放位置附近，用于精细拖动、看网格、看 cue、调网格。

### 4.3 网格线

网格线表示歌曲 beat grid。

用户看到的网格线包括：

- 普通 beat line。
- 更醒目的 bar line。

网格线的位置代表当前保存的歌曲节拍网格。除非用户执行真实网格编辑操作，否则网格线不应移动。

补充说明：

- 网格线的数据来源始终是歌曲真实 beat grid。
- 用户执行左右移动网格线、BPM 输入、Set Bar Line 等网格编辑操作时，修改的都是歌曲真实 beat grid，并且应保存。
- Beat Sync 开启时，当前双轨画面还必须保证两轨视觉网格保持对齐。因此，保存目标轨真实网格后，系统需要通过当前同步播放或相位补偿，让用户继续看到两轨网格对齐。
- 不能把“当前画面网格保持对齐”理解成“不保存真实网格”。

### 4.4 播放头

播放头表示当前听到的位置。

要求：

- 播放头位置必须和声音一致。
- 播放头位置必须和大波形当前中心/参考位置一致。
- seek、拖动、cue、beat jump 后，播放头必须到达用户操作的目标位置。

### 4.5 Cue 与 Loop 标记

可见标记包括：

- Hot Cue。
- Memory Cue。
- Loop 起止范围。

这些标记显示在歌曲时间线上。用户调整同步或移动歌曲相位时，不应让 cue 标记自己漂走。

## 5. 双轨模式总览

双轨模式用于两首歌同时浏览、播放、同步、混音。

双轨模式应该显示：

- 上轨播放控制。
- 下轨播放控制。
- 上轨概览波形。
- 下轨概览波形。
- 上轨大波形。
- 下轨大波形。
- 中间 crossfader 面板。
- 双轨连接同步按钮。
- 右侧 cue 面板。

双轨模式不应该显示编辑模式专用的上一首/下一首/beat jump 控制条。

### 5.1 进入双轨模式

用户进入双轨模式时：

- 如果上轨已经有歌，上轨显示这首歌的波形、网格、cue、loop、播放状态。
- 如果下轨已经有歌，下轨显示这首歌的波形、网格、cue、loop、播放状态。
- 如果某条轨道没有歌，该轨道显示空状态。
- Crossfader 显示当前混音位置。
- 双轨连接同步按钮显示当前开启或关闭状态。

不应该发生：

- 进入双轨模式导致歌曲被卸载。
- 进入双轨模式导致播放位置重置。
- 进入双轨模式导致 crossfader 自动回中。
- 进入双轨模式导致 Beat Sync 自动关闭。

### 5.2 双轨模式中两轨的关系

上轨和下轨是对称的。

要求：

- 上轨能做的常规操作，下轨也应该能做。
- 操作上轨不应该误改下轨。
- 操作下轨不应该误改上轨。
- 只有明确的双轨操作才可以同时影响两轨，例如 crossfader、双轨连接同步。

## 6. 编辑模式总览

编辑模式用于单首歌精细查看和编辑。

编辑模式应该显示：

- 编辑模式专用控制条。
- 上轨播放控制。
- 上轨概览波形。
- 上轨大波形。

编辑模式不应该显示：

- 下轨播放控制。
- 下轨概览波形。
- 下轨大波形。
- Crossfader 面板。
- 双轨连接同步按钮。
- 右侧双轨 cue 面板。

### 6.1 进入编辑模式

用户进入编辑模式时：

- 只看到上轨的工作区。
- 上轨已有歌曲继续显示。
- 上轨播放状态保持。
- 上轨大波形使用更适合精细编辑的显示方式。

不应该发生：

- 下轨歌曲被清空。
- 下轨 cue 或网格被清空。
- Crossfader 值被重置。
- 双轨模式状态被永久改坏。

### 6.2 退出编辑模式

用户退出编辑模式回到双轨模式时：

- 上轨状态保留。
- 下轨重新显示。
- Crossfader 面板重新显示。
- 双轨连接同步按钮重新显示。
- 下轨回到进入编辑模式前应有的歌曲、网格、cue、loop 和播放状态。

不应该发生：

- 回到双轨后下轨变空。
- 回到双轨后两轨网格错乱。
- 回到双轨后 sync 状态被无故取消。

## 7. 加载与卸载歌曲

### 7.1 向空轨道加载歌曲

用户把歌曲加载到空轨道时：

- 轨道显示歌曲名和相关信息。
- 概览波形开始加载并显示。
- 大波形开始加载并显示。
- 已有网格线显示出来。
- 已有 hot cue、memory cue、loop 标记显示出来。
- 播放按钮变为可用。
- Cue、loop、grid、quantize、master tempo 等歌曲相关按钮按状态变为可用。

声音表现：

- 加载歌曲本身不应该自动播放，除非用户之前明确触发了“加载后播放”的操作。
- 没有播放时不应突然出声。

### 7.2 替换已有歌曲

用户把新歌曲加载到已经有歌的轨道时：

- 旧歌曲的波形、网格、cue、loop 从该轨道消失。
- 新歌曲的波形、网格、cue、loop 显示出来。
- 该轨道播放位置切换到新歌曲的默认位置。
- 另一轨不受影响。

不应该发生：

- 新歌短暂显示旧歌的网格线。
- 新歌短暂显示旧歌的 cue。
- 替换上轨导致下轨停止。
- 替换下轨导致上轨停止。

### 7.3 卸载歌曲

用户卸载某轨歌曲时：

- 该轨回到空状态。
- 该轨播放停止。
- 该轨波形、网格、cue、loop 清空。
- 该轨相关按钮禁用。
- 另一轨继续保持原状态。

如果卸载的是当前 sync leader：

- Sync 状态应进入清晰的不可同步或关闭状态。
- UI 不应继续显示一个已经不存在的 leader。

## 8. 播放控制

### 8.1 Play / Pause

用户点击某轨播放按钮：

- 如果该轨暂停，则开始播放。
- 如果该轨播放中，则暂停。
- 播放按钮状态立即更新。
- 播放头开始或停止移动。
- 大波形运动状态和声音一致。

不应该发生：

- 按钮显示播放但没有声音。
- 有声音但按钮显示暂停。
- 播放头不动但声音在播。
- 声音停了但播放头还在继续跑。

### 8.2 Cue 按住预听

用户按住 Cue：

- 如果该轨有歌曲，开始从 cue 点或当前规则定义的位置预听。
- 松开 Cue 后，如果没有进入持续播放状态，应回到 cue 行为对应的位置。
- Cue active 状态应在按住期间显示。

不应该发生：

- 按住上轨 Cue 触发下轨声音。
- 松开 Cue 后播放状态和按钮状态不一致。

### 8.3 播放中切换模式

如果用户在播放中从双轨切到编辑模式：

- 上轨播放状态保持。
- 如果产品当前行为允许下轨继续播放，下轨声音应继续但界面隐藏。
- 如果产品当前行为会停止或处理下轨，必须表现稳定，不能像崩了一样半停半播。

如果用户在播放中从编辑模式回到双轨模式：

- 可见界面应追上真实播放状态。
- 不应出现声音已经跑到后面、播放头还停在旧位置的情况。

## 9. 概览波形操作

### 9.1 点击概览波形跳转

用户点击某轨概览波形：

- 该轨播放头跳到点击位置。
- 该轨大波形切换到点击位置附近。
- 如果该轨正在播放，声音从新位置继续播放。
- 如果该轨暂停，声音不自动播放。
- 另一轨不受影响，除非开启了明确的同步跟随规则。

不应该发生：

- 点击上轨概览导致下轨跳转。
- 点击下轨概览导致上轨跳转。
- 大波形长时间黑屏。
- 播放头到了新位置但声音还在旧位置。
- 声音到了新位置但波形还显示旧位置。

### 9.2 概览波形中的标记

概览波形应显示：

- 当前播放头。
- 当前大波形可见范围。
- 网格线。
- Hot Cue。
- Memory Cue。
- Loop 范围。

要求：

- 标记位置跟歌曲时间一致。
- 第一拍附近的 cue 标签不能半截跑出波形框。
- 日间模式和夜间模式都能清楚看见标记。

### 9.3 概览波形加载中

概览波形加载中时：

- 可以显示加载状态或渐进显示。
- 已经可用的数据可以先显示。
- 不应该显示上一首歌残留波形。
- 不应该因为加载中阻塞播放控制。

## 10. 大波形操作

### 10.1 播放中的大波形

播放时：

- 播放头保持稳定。
- 波形平滑移动。
- 网格线相对播放头的运动稳定。
- Cue 和 loop 标记跟随歌曲时间正确经过播放头。

不应该发生：

- 波形抖动。
- 播放头忽快忽慢。
- 新数据加载时整个大波形闪一下。
- 播放中因为补画导致位置跳一下。

### 10.2 拖动大波形

用户拖动某轨大波形：

- 该轨大波形应跟手移动。
- 拖动过程中播放头位置预览应清晰。
- 如果有拖动预听，听到的位置应跟拖动位置一致。
- 松手后，该轨播放位置落到松手目标位置。
- 播放中拖动后，应从新位置继续播放。
- 暂停中拖动后，应保持暂停，只更新位置。

不应该发生：

- 鼠标拖动没反应但声音变了。
- 波形动了但松手后声音回到旧位置。
- 声音动了但波形停在旧位置。
- 拖动上轨影响下轨位置。

### 10.3 拖到歌曲起点之前

在支持负时间线的大波形里，用户可以把大波形拖到歌曲起点之前。

松手后：

- 位置保持在起点之前，不自动弹回 0。
- 播放时起点之前是静音空白。
- 播放速度正常。
- 播放头走到歌曲实际开始处后，声音自然进入。

不应该发生：

- 松手后自动吸回 0。
- 空白区域播放速度异常。
- 到 0 秒时明显卡一下。

### 10.4 大波形缩放

用户调整大波形 zoom：

- 当前关注位置保持稳定。
- 波形细节随 zoom 增减。
- 网格线间距随视图比例正确变化。
- Cue 和 loop 标记保持在对应歌曲位置。

不应该发生：

- 缩放后播放位置跳走。
- 缩放后网格和 cue 对不上。
- 编辑模式缩放影响双轨模式不该共享的 zoom 状态。

## 11. Beat Sync

### 11.1 Beat Sync 按钮状态

Beat Sync 按钮可能表现为：

- 未开启。
- 已开启。
- 闪烁或提示态，表示只同步速度或等待完整对齐。

要求：

- 按钮状态必须和真实同步状态一致。
- 用户开启 sync 后，不能无原因自己变成关闭。
- 用户关闭 sync 后，不能继续显示开启。

### 11.2 开启 Beat Sync

用户对某轨开启 Beat Sync：

- 如果另一轨可作为参考轨，当前轨向参考轨对齐。
- 对齐目标是最近的 beat 网格线。
- 最近的 beat 网格线不是固定 32 拍，不是固定小节线，不是 phrase 起点。
- 对齐后，两轨节拍应听起来对齐。
- UI 上 sync 状态显示开启或对应提示态。

不应该发生：

- 开启 sync 后明显对到错误的远处网格线。
- 开启 sync 后播放位置被旧界面状态拉回。
- 开启 sync 后按钮亮了但声音没有同步。
- 开启 sync 后声音同步了但波形/播放头状态乱掉。

### 11.3 关闭 Beat Sync

用户关闭某轨 Beat Sync：

- 该轨停止跟随另一轨同步。
- 该轨当前播放位置保持。
- 该轨当前播放速度按产品既有规则保持或恢复。
- 另一轨不应被停止。

不应该发生：

- 关闭上轨 sync 导致下轨跳转。
- 关闭下轨 sync 导致上轨跳转。
- 关闭 sync 清空网格或 cue。

### 11.4 Sync Leader / Master

用户切换 master：

- 被设为 master 的轨道成为同步参考。
- 另一轨如开启 sync，应跟随新的参考轨。
- 切换 master 不应改变任意歌曲的真实网格。
- 切换 master 不应清空 cue、loop、phase 调整或播放状态。

不应该发生：

- 切换 master 后两轨视觉网格突然错位。
- 切换 master 后 follower 跳到明显错误位置。
- 切换 master 后 sync 按钮状态和声音状态不一致。

## 12. 左右移动网格线按钮

这里的按钮包括：

- 左移大步。
- 左移小步。
- 右移小步。
- 右移大步。

当前表现标准：

- 小步：5ms。
- 大步：20ms。

### 12.1 Beat Sync 关闭时点击

前提：

- 目标轨道有歌曲。
- 目标轨道 Beat Sync 关闭。

用户点击左移按钮：

- 目标轨道网格线向左移动。
- 波形内容不反向移动。
- 播放头位置不因为网格保存而跳走。
- 如果正在播放，声音继续从当前位置播放，不应突然跳变。
- 该修改作为真实网格编辑保存。

用户点击右移按钮：

- 目标轨道网格线向右移动。
- 波形内容不反向移动。
- 播放头位置不因为网格保存而跳走。
- 如果正在播放，声音继续从当前位置播放，不应突然跳变。
- 该修改作为真实网格编辑保存。

不应该发生：

- Beat Sync 关闭时点击按钮却只移动波形、不移动网格。
- 修改上轨网格导致下轨真实网格被改。
- 修改下轨网格导致上轨真实网格被改。

### 12.2 Beat Sync 开启时点击

前提：

- 目标轨道有歌曲。
- 目标轨道 Beat Sync 开启。
- 用户点击左移或右移网格线按钮。

Beat Sync 开启时，左右移动网格线按钮仍然是一次真实网格编辑和保存。区别是当前双轨同步画面必须保持两轨视觉网格对齐，所以保存后需要立即对目标轨做同步相位补偿，让用户看到波形和声音相对网格移动，而不是看到两轨网格漂开。

用户点击后必须发生：

- 目标轨真实网格按按钮方向移动并保存。
- 当前双轨 Sync 画面里，目标轨和参考轨的视觉网格仍然保持对齐。
- 目标轨波形内容相对当前画面中仍然对齐的网格线发生移动。
- 目标轨声音跟随波形发生同方向变化。
- 目标轨道 Beat Sync 按钮仍保持开启状态。
- 如果原来是完整 sync，不应因为这次点击自动变成关闭。
- 用户不需要点任何额外保存按钮。

用户点击左移按钮时：

- 目标轨真实网格向左移动并保存。
- 当前 Sync 画面为了维持两轨网格对齐，目标轨歌曲内容、波形和声音表现为向右发生等效移动。

用户点击右移按钮时：

- 目标轨真实网格向右移动并保存。
- 当前 Sync 画面为了维持两轨网格对齐，目标轨歌曲内容、波形和声音表现为向左发生等效移动。

补偿生命周期：

- 同步补偿只服务于当前这次 Beat Sync 画面继续稳定对齐，不是一个需要用户单独保存的新设置。
- 换歌、卸载、重新加载歌曲后，应直接使用已经保存过的新真实网格，不应依赖旧的同步补偿。
- 用户关闭 Beat Sync 时，当前播放位置不应突然跳变；关闭后按已经保存的新真实网格继续显示和操作。
- 用户重新开启 Beat Sync 时，应基于已经保存的新真实网格重新建立同步关系。

目标轨是 Sync Leader / Master 时：

- 无论目标轨是 follower 还是 leader / master，左右移动按钮都作用于用户点击的那一轨真实网格，并按按钮方向保存。
- 如果目标轨是 follower，保存 follower 真实网格后，当前 Sync 画面需要补偿 follower，让它继续和 leader 对齐。
- 如果目标轨是 leader / master，保存 leader 真实网格后，当前 Sync 关系需要整体保持稳定，不能让 follower 视觉网格漂开、声音错开或 Sync 状态掉线。
- 不允许因为用户修改了 leader / master 的真实网格，就静默关闭 Beat Sync。

方向示例：

- 小步为 5ms 时，点击右移小步，目标轨真实网格向右保存 5ms；当前 Sync 画面里，目标轨波形和声音表现为向左等效移动 5ms。
- 小步为 5ms 时，点击左移小步，目标轨真实网格向左保存 5ms；当前 Sync 画面里，目标轨波形和声音表现为向右等效移动 5ms。
- 大步 20ms 同理，真实网格按按钮方向保存 20ms，当前 Sync 画面里的波形和声音按反方向等效补偿 20ms。

不应该发生：

- 真实网格已经保存，但当前 Sync 画面没有做同步补偿，导致两轨视觉网格漂开。
- 只做了当前 Sync 补偿，却没有保存真实网格，导致下次打开修正丢失。
- 波形移动了但声音没移动。
- 声音移动了但波形没移动。
- Beat Sync 按钮变成关闭。
- 进入闪烁状态却没有明确原因。
- 弹出“保存为新的网格”。

### 12.3 连续点击

用户连续点击四个左右移动按钮：

- 每次点击都按当前步长累积。
- 界面每次都立即反馈。
- 声音每次都跟随变化。
- Beat Sync 状态持续保持。
- 不应因为快速点击造成按钮状态乱跳。

不应该发生：

- 第一次对，第二次开始方向反了。
- 连点后网格线慢慢漂开。
- 连点后声音和波形累计量不一致。

### 12.4 播放中点击

播放中点击左右移动按钮：

- 不能爆音。
- 不能明显卡顿。
- 不能让播放头突然跳一大截。
- 用户应听到歌曲相位细微变化。
- 用户应看到波形相对网格细微变化。

### 12.5 暂停中点击

暂停中点击左右移动按钮：

- 界面立即更新。
- 播放头保持在当前 timeline 位置。
- 下次播放时，声音从调整后的歌曲相位开始。
- Beat Sync 状态保持。

## 13. BPM 输入与 Tap

### 13.1 输入 BPM

用户修改某轨 BPM 输入：

- 该轨网格线间距变化。
- BPM 显示为格式化后的值。
- 修改作为真实网格编辑保存。
- 该轨波形内容不应该因为 BPM 输入被当成相位移动。

如果该轨 Beat Sync 关闭：

- 只影响该轨真实网格。
- 不影响另一轨。

如果该轨 Beat Sync 开启：

- 网格间距变化后，sync 状态应保持可理解。
- 如果仍能保持同步，按钮保持开启。
- 如果不能保持完整同步，应显示明确的提示态，而不是静默关闭。

不应该发生：

- 输入 BPM 后 sync 按钮无原因关闭。
- 输入 BPM 后声音和播放头突然跳到不相关位置。
- 输入上轨 BPM 改了下轨 BPM。

### 13.2 BPM Tap

用户 tap BPM：

- 系统根据连续点击估算 BPM。
- BPM 输入框更新。
- 网格线间距更新。
- 修改作为真实网格编辑保存。

不应该发生：

- tap 后只改显示、不改网格。
- tap 后误触发播放跳转。
- tap 后清空 cue 或 loop。

## 14. Set Bar Line

### 14.1 点击 Set Bar Line

用户点击“Set Bar Line at Playhead”：

- 当前播放头位置成为 bar line 对齐参考。
- 该轨真实网格更新。
- 网格线显示立即变化。
- 修改保存。

如果正在播放：

- 播放继续。
- 声音不应跳变。
- 播放头不应突然离开当前位置。

如果 Beat Sync 开启：

- 这是一次真实网格编辑。它和左右移动按钮一样必须保存真实网格，但不应被误处理成只作用于当前 Sync 画面的临时补偿。
- Sync 状态应根据新的真实网格保持或进入明确提示态。
- 不应静默关闭 sync。

### 14.2 Bar Line Picking

用户开启 bar line picking：

- 对应按钮进入 active/cancel 状态。
- 用户再选择位置后，该位置成为 bar line 对齐参考。
- 选择完成后退出 picking 状态。

用户取消 picking：

- 不修改网格。
- 按钮回到普通状态。

不应该发生：

- picking 状态影响另一轨。
- 取消后仍然处于 picking 状态。
- picking 时误触发播放跳转，除非该交互本来就是当前产品规则。

## 15. Hot Cue

### 15.1 Hot Cue 面板

双轨模式右侧显示 top/bottom 两组 hot cue 或 memory cue 面板。

Hot Cue 面板要求：

- 每个 slot 显示 label。
- 有值 slot 显示时间和颜色。
- 无值 slot 显示空状态。
- loop hot cue 显示 loop 标识。

### 15.2 点击空 Hot Cue

用户点击空 hot cue slot：

- 按当前产品规则创建 hot cue，或执行当前已有的空 slot 行为。
- 新 hot cue 位置取当前播放头位置。
- 新 hot cue 显示在概览波形、大波形和 cue 面板。

不应该发生：

- 创建到另一轨。
- 创建位置和当前播放头不一致。
- 创建后播放位置跳走。

### 15.3 点击已有 Hot Cue

用户点击已有 hot cue：

- 该轨跳转到 hot cue 位置。
- 播放头更新。
- 大波形更新。
- 如果播放中，从 hot cue 位置继续播放。
- 如果暂停中，跳到 hot cue 位置但不自动播放，除非现有产品规则就是会播放。

不应该发生：

- 点击 top hot cue 跳 bottom。
- 点击 bottom hot cue 跳 top。
- 声音去了新位置但波形没去。

### 15.4 删除 Hot Cue

用户点击 hot cue 删除按钮：

- 只删除对应轨道、对应 slot。
- 概览波形、大波形、cue 面板同步移除该标记。
- 其他 slot 不受影响。

不应该发生：

- 删除上轨 hot cue 影响下轨。
- 删除一个 slot 导致其他 slot 重排错乱。

## 16. Memory Cue

### 16.1 Memory Cue 面板

Memory Cue 面板显示：

- cue 时间。
- cue 颜色或标识。
- loop memory cue 的 loop 标识。
- 删除按钮。

没有 memory cue 时：

- 面板显示空列表状态。
- 布局不塌陷。

### 16.2 创建 Memory Cue

用户点击创建 memory cue：

- 在当前播放头位置创建 memory cue。
- 标记显示在概览波形、大波形和 memory cue 面板。
- 播放状态不应被改变。

### 16.3 点击 Memory Cue

用户点击 memory cue：

- 该轨跳转到 memory cue 位置。
- 播放头、大波形、声音位置一致。
- 播放中继续播放。
- 暂停中保持暂停。

### 16.4 删除 Memory Cue

用户点击 memory cue 删除：

- 只删除对应轨道、对应 memory cue。
- 其他 cue 不受影响。
- 删除后波形上的标记同步消失。

## 17. Loop

### 17.1 Loop 按钮

Loop 控件包括：

- Loop 缩短。
- Loop 加长。
- Loop 开关。
- Loop 当前长度显示。

没有歌曲时：

- Loop 控件禁用。

有歌曲时：

- Loop 控件可用。

### 17.2 开启 Loop

用户开启 loop：

- 当前轨道进入 loop 状态。
- Loop 范围显示在波形上。
- 播放到 loop 末尾时回到 loop 起点。
- Loop 按钮显示 active。

不应该发生：

- 开启上轨 loop 导致下轨 loop 开启。
- Loop 范围显示和实际回跳位置不一致。

### 17.3 关闭 Loop

用户关闭 loop：

- 当前轨道退出 loop。
- Loop active 状态消失。
- 已有 loop 标记按当前产品规则保留或隐藏。
- 播放继续向后走。

### 17.4 调整 Loop 长度

用户点击 loop 缩短或加长：

- 当前 loop 长度按步进变化。
- Loop 范围在波形上同步变化。
- 如果正在 loop 中，下一次回跳按新范围执行。

不应该发生：

- 调整 loop 长度修改 beatgrid。
- 调整上轨 loop 影响下轨。

## 18. Quantize

### 18.1 开关 Quantize

用户点击 Q：

- 当前轨道 quantize 开启或关闭。
- Q 按钮 active 状态更新。

Quantize 开启时：

- 需要吸附的 cue、loop、seek 或相关操作按网格吸附。
- 吸附参考用户看到的视觉网格线。

Quantize 关闭时：

- 对应操作不强制吸附到网格。

不应该发生：

- 开启 Q 后按钮不亮。
- 关闭 Q 后仍按开启状态吸附。
- 上轨 Q 影响下轨 Q。

## 19. Master Tempo 与 Reset Tempo

### 19.1 Master Tempo

用户点击 MT：

- 当前轨道 master tempo 开启或关闭。
- MT active 状态更新。
- 当前播放位置不变。
- 当前网格不变。
- 当前 cue 和 loop 不变。
- Beat Sync 不应无故关闭。

播放中切换 MT：

- 声音应稳定。
- 不应爆音。
- 不应明显卡顿。

### 19.2 Reset Tempo

用户点击 RES：

- 当前轨道播放速度恢复到基准速度。
- 播放位置不变。
- 网格不变。
- Cue、loop、hot cue、memory cue 不变。

如果当前轨道 Beat Sync 开启：

- Reset 后 sync 状态必须清楚。
- 如果仍跟随参考轨，按钮保持开启。
- 如果进入提示态，必须是可解释的提示态。
- 不应静默关闭 sync。

## 20. Metronome

### 20.1 开启 Metronome

用户开启某轨 metronome：

- 该轨播放经过网格线时响。
- 节拍声音应跟播放位置和网格线一致。
- 按钮显示开启状态。

不应该发生：

- 没经过网格线也乱响。
- 经过网格线不响。
- 上轨 metronome 影响下轨。
- metronome 跟屏幕刷新走，导致声音和播放不准。

### 20.2 关闭 Metronome

用户关闭 metronome：

- 该轨 metronome 不再响。
- 播放不受影响。
- 网格不受影响。

### 20.3 调整 Metronome 音量

用户循环 metronome 音量：

- metronome 音量按档位变化。
- 其他歌曲声音音量不应改变。
- 另一轨 metronome 音量不应改变。

## 21. Crossfader 与音量混合

### 21.1 拖动 Crossfader

用户拖动 crossfader：

- 靠近上轨一侧时，上轨声音更明显，下轨声音更小。
- 靠近下轨一侧时，下轨声音更明显，上轨声音更小。
- 居中时，两轨按当前混音规则同时输出。
- Thumb 位置和实际声音混合一致。

不应该发生：

- Thumb 在上轨侧但听到下轨更大。
- 声音变化了但 thumb 不动。
- Thumb 动了但声音不变。

### 21.2 展开/收起 Fader 控制

用户展开 fader 控制：

- 显示更多双轨控制，例如双轨连接同步按钮。
- 原有播放、网格、sync、cue 状态保持。

用户收起 fader 控制：

- 隐藏扩展控制。
- 原有状态保持。

不应该发生：

- 展开/收起导致播放停止。
- 展开/收起导致 sync 关闭。
- 展开/收起导致 crossfader 回中。

### 21.3 Deck Band 开关

用户切换某轨 band：

- 只影响该轨对应频段输出。
- UI active 状态同步变化。
- 另一轨 band 不受影响。

## 22. 编辑模式专用操作

### 22.1 载入上一首

用户点击上一首：

- 如果存在上一首，上轨加载上一首。
- 上轨波形、网格、cue、loop 更新为上一首。
- 如果不存在上一首，按钮禁用或不可操作。

不应该发生：

- 点击上一首影响下轨。
- 没有上一首时仍然触发异常加载。

### 22.2 载入下一首

用户点击下一首：

- 如果存在下一首，上轨加载下一首。
- 上轨波形、网格、cue、loop 更新为下一首。
- 如果不存在下一首，按钮禁用或不可操作。

不应该发生：

- 点击下一首影响下轨。
- 没有下一首时仍然触发异常加载。

### 22.3 选择 Beat Step

用户选择 beat step：

- 可选值为 4、8、16、32、128 beats。
- 当前选择值显示在控件上。
- 后续前进/后退 beat jump 使用该步长。

没有歌曲时：

- beat step 选择禁用。

### 22.4 后退 N Beats

用户点击后退 N beats：

- 上轨播放位置向前回退 N 个 beat。
- 大波形和播放头更新。
- 如果正在播放，从新位置继续播放。
- 如果暂停中，停在新位置。
- 不修改网格。
- 不修改 cue。
- 不修改 loop。

### 22.5 前进 N Beats

用户点击前进 N beats：

- 上轨播放位置向后前进 N 个 beat。
- 大波形和播放头更新。
- 如果正在播放，从新位置继续播放。
- 如果暂停中，停在新位置。
- 不修改网格。
- 不修改 cue。
- 不修改 loop。

不应该发生：

- beat jump 导致网格移动。
- beat jump 导致下轨跳转。
- beat jump 后声音位置和播放头不一致。

## 23. 空状态、禁用状态与加载状态

### 23.1 没有歌曲

某轨没有歌曲时：

- 播放按钮禁用或不可触发。
- Cue 禁用或不可触发。
- Grid 调整禁用。
- BPM 输入禁用。
- Loop 禁用。
- Quantize 禁用。
- Master Tempo 禁用。
- Move / eject 等按产品规则显示。

不应该发生：

- 空轨道点击播放后报错。
- 空轨道点击 grid 调整后影响上一首歌。
- 空轨道显示残留 cue。

### 23.2 波形加载中

波形加载中时：

- 用户能看出正在加载。
- 已加载部分可以显示。
- 播放控制不应因为视觉加载中完全卡死，除非音频本身还不可用。
- 加载完成后替换为清晰波形。

不应该发生：

- 长时间黑屏没有恢复。
- 显示旧歌曲波形。
- 加载完成后播放头错位。

### 23.3 操作不可用

某操作不可用时：

- 按钮禁用或表现为不可操作。
- 鼠标悬停说明原因。
- 点击不会产生半截状态变化。

## 24. 日间模式与夜间模式

所有双轨和编辑模式界面都必须同时适配日间模式与夜间模式。

要求：

- 波形背景清晰。
- 网格线清晰。
- Cue 标记清晰。
- Loop 范围清晰。
- 按钮 active / disabled / hover 状态清晰。
- 文本不挤出按钮。
- 小标签不被裁掉。

不应该发生：

- 日间模式下波形直接贴白底看不清。
- 夜间模式下网格线几乎不可见。
- 按钮文字溢出。
- cue 标签遮挡重要内容。

## 25. 典型验收场景

### 25.1 双轨无 Sync 调网格

操作：

1. 上轨加载歌曲。
2. 下轨加载歌曲。
3. 确认上轨 Beat Sync 关闭。
4. 点击上轨左移小步。

预期：

- 上轨网格线向左移动 5ms。
- 上轨波形不反向移动。
- 下轨不变。
- 声音不跳。

继续操作：

1. 点击上轨右移大步。

预期：

- 上轨网格线向右移动 20ms。
- 下轨不变。
- 修改保存为上轨真实网格。

### 25.2 双轨有 Sync 调网格按钮

操作：

1. 上轨加载歌曲。
2. 下轨加载歌曲。
3. 开启 Beat Sync，让两轨对齐。
4. 对 follower 点击右移小步。

预期：

- 当前画面里的 follower 视觉网格线不漂开。
- 当前画面里的 leader 视觉网格线不漂开。
- 两轨视觉网格仍对齐。
- follower 真实网格向右移动 5ms 并保存。
- follower 波形向左移动 5ms 的等效量。
- follower 声音跟着波形变化。
- Beat Sync 仍开启。
- 不弹额外保存流程。

继续操作：

1. 对 follower 点击左移大步。

预期：

- 当前画面里的 follower 视觉网格线仍不漂开。
- follower 真实网格向左移动 20ms 并保存。
- follower 波形向右移动 20ms 的等效量。
- 声音跟着移动。
- Beat Sync 仍开启。

### 25.3 播放中 Sync 调整

操作：

1. 两轨播放。
2. Beat Sync 开启。
3. 点击 follower 左移/右移按钮。

预期：

- 音频不中断。
- 不爆音。
- 不明显卡顿。
- 目标轨真实网格按按钮方向保存。
- 波形和声音一起发生细微相位变化。
- 两轨视觉网格仍对齐。
- Sync 不掉。

### 25.4 暂停中 Sync 调整

操作：

1. 两轨暂停。
2. Beat Sync 开启。
3. 点击 follower 左移/右移按钮。
4. 再点击播放。

预期：

- 暂停时界面已经显示波形相位变化。
- 目标轨真实网格按按钮方向保存。
- 播放后声音从调整后的相位开始。
- 网格仍对齐。
- Sync 仍开启。

### 25.5 概览跳转

操作：

1. 某轨播放中。
2. 点击该轨概览波形中后段。

预期：

- 播放头跳到点击位置。
- 大波形切到点击位置附近。
- 声音从点击位置继续播放。
- 大波形不长期黑屏。

### 25.6 大波形拖动

操作：

1. 某轨播放中。
2. 拖动大波形到新位置。
3. 松手。

预期：

- 拖动过程波形跟手。
- 松手后播放头、大波形、声音一致。
- 正在播放则从新位置继续播放。

### 25.7 编辑模式 Beat Jump

操作：

1. 进入编辑模式。
2. 上轨加载歌曲。
3. 选择 16 beats。
4. 点击前进。

预期：

- 上轨播放位置前进 16 beats。
- 大波形和播放头更新。
- 网格不变。
- 下轨不受影响。

### 25.8 Cue 操作

操作：

1. 双轨模式，上轨加载歌曲。
2. 点击上轨 hot cue。

预期：

- 上轨跳到 hot cue 位置。
- 上轨播放头、大波形、声音一致。
- 下轨不跳。

### 25.9 Loop 操作

操作：

1. 上轨播放中。
2. 开启 loop。
3. 调整 loop 长度。

预期：

- 上轨显示 loop 范围。
- 上轨按范围循环。
- 调整长度后范围变化。
- 下轨不受影响。

### 25.10 模式切换

操作：

1. 双轨模式下两轨都有歌曲。
2. 进入编辑模式。
3. 再回到双轨模式。

预期：

- 编辑模式只显示上轨。
- 回到双轨后下轨仍在。
- Crossfader 状态仍在。
- Sync 状态不被无故取消。

## 26. 绝对不能出现的表现

- 用户点左右移动网格线，Beat Sync 自动变成关闭。
- Beat Sync 开启时，左右移动按钮导致两轨视觉网格不再对齐。
- 波形移动但声音不移动。
- 声音移动但波形不移动。
- 网格移动但用户预期它不动。
- 用户没有要求额外保存流程，却弹出保存为新网格之类的额外流程。
- 编辑模式影响下轨数据。
- 概览 seek 后大波形长时间黑屏。
- 播放中出现明显卡顿或爆音。
- 按钮状态和真实播放状态不一致。
- 上轨操作误伤下轨。
- 下轨操作误伤上轨。
- 日间模式或夜间模式出现看不清、裁切、文字溢出。
