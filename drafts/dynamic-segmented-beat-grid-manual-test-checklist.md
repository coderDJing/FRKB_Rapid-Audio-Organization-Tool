# 动态多 clip 网格功能验收速查

创建日期：2026-07-05

这份文档只列本轮需要看的功能面，避免测试漏项；完整语义见 `drafts/dynamic-segmented-beat-grid-design-draft.md`。

## 1. 固定 BPM 旧路径

- [ ] 固定 BPM 歌曲的显示、播放、seek、cue、loop、beat jump、metronome 仍走原逻辑。
- [ ] 固定 BPM 歌曲不出现动态 clip 边界，也不会被无故保存成动态网格。

## 2. 编辑模式动态网格

- [ ] 点击“从当前点之后”只创建 / 选中 clip 边界，不启动前台 Key / BPM 分析。
- [ ] “整首调整”和“从当前点之后”是互斥状态按钮，位于大幅右移按钮后面；默认选中“整首调整”。
- [ ] 新建边界与播放指示线重合时，仍能看到 clip 边界标记。
- [ ] BPM、tap BPM、左移 / 右移只影响当前选中的 clip，右段拍线不会画到边界左侧。
- [ ] 点击“整首调整”能退出后段编辑视图，左侧网格恢复显示。
- [ ] 删除边界后能合并 clip；合并成单一网格时能退回固定 BPM。

## 3. 波形显示

- [ ] 大波形按完整动态网格显示拍线，跨 clip 后间距自然变密 / 变疏。
- [ ] 动态 clip 边界载入、播放、暂停位置连续，经过边界不卡顿；边界是顶格极简竖线，点击后有选中加亮和删除入口。
- [ ] 概览 / 小波形只显示轻量边界提示，不铺满密集动态网格。
- [ ] 暗色和亮色主题下，动态网格、边界、选中态、播放头都清楚可见。

## 4. 列表和共享

- [ ] 动态歌曲 BPM 显示多段摘要，筛选能命中任意 clip，排序按最低 clip BPM。
- [ ] 普通歌单、Set、Mixtape 内部引用能保留并共享同一份 `beatGridMap`。
- [ ] 清缓存 / 重新扫描不应无故丢失手工动态网格；主动重新分析除外。

## 5. 播放和同步

- [ ] 动态歌曲普通播放跨 clip 不 seek、不跳位。
- [ ] Quantize、Beat Jump、Loop、Metronome 使用动态全局拍线 / 累计 beat。
- [ ] 动态歌参与 BeatSync 时使用局部 BPM 和动态 beat ordinal，固定歌与动态歌混合时相位稳定。

## 6. 分析链路

- [ ] 动态网格变化后，段落结果可按网格签名待更新，但旧段落视觉不闪烁、不消失。
- [ ] Key 分析不受动态网格影响；只有用户主动重新分析 BPM / 网格时才覆盖动态网格。

## 7. Mixtape

- [ ] 动态歌曲进入 Mixtape 后使用完整 `SongBeatGridMap`，不退化成第一段 BPM。
- [ ] Mixtape 预览、导出、snap、beat align、loop 展开使用同一套动态时间映射。
- [ ] Mixtape 全局 BPM 自动贡献点能跟随动态歌曲移动 / 删除，手动 BPM 点优先。

## 8. 不属于本轮

- Rekordbox beat grid 写回不属于本轮。
- 自动分析不识别、不生成动态 BPM。
