# 双轨横推模式快捷键草案

## 背景

双轨横推模式不应再照搬主播放器的“按秒快进 / 快退”思路。
这一模式更接近双 deck 预听与整理场景，核心需求是：

- 快速控制上下轨播放
- 快速执行上下轨 `CUE`
- 按小节跳转定位
- 快速把当前轨曲目移到筛选库 / 精选库
- 快速删除当前轨曲目
- 直接用键盘控制 crossfader

本草案以“尽量保留主播放器已有肌肉记忆”为前提，但不强行追求所有键位一比一复用。

## 设计原则

1. `Shift` 只负责区分上下轨，不额外承担别的模式切换职责。
2. 高频操作优先落在左手区域或方向键上，减少键位分裂。
3. 双轨模式不再提供“按秒快进 / 快退”快捷键，只保留“按小节跳转”。
4. 删除行为沿用主播放器逻辑：普通库中直接移到回收站；仅在回收站中才弹彻底删除确认。
5. 不为低频或容易误触的操作强行加快捷键。

## 快捷键定义

### 轨道播放

- `Space`：上轨播放 / 暂停
- `Shift+Space`：下轨播放 / 暂停

### CUE

- `C`：上轨 `CUE`
- `Shift+C`：下轨 `CUE`

`CUE` 行为必须与当前 UI 按钮完全一致：

- `keydown` 按下时触发
- 支持按住不松手进行预听
- `keyup` 松手后返回 cue 点
- 不能简化成一次性 click 语义

### 按小节跳转

- `A` / `Left`：上轨退一小节
- `D` / `Right`：上轨进一小节
- `Shift+A` / `Shift+Left`：下轨退一小节
- `Shift+D` / `Shift+Right`：下轨进一小节

说明：

- 跳转单位为“一小节”
- 具体秒数按当前轨有效 BPM / grid 计算
- 若轨道当前正在播放，则 seek 后继续播放
- 若轨道当前处于暂停，则只更新播放位置

### Crossfader

- `W` / `Up`：crossfader 往上移动
- `S` / `Down`：crossfader 往下移动
- `Shift+W`：crossfader 回正
- `Shift+S`：crossfader 回正
- `Shift+Up`：crossfader 回正
- `Shift+Down`：crossfader 回正

交互要求：

- 按一下移动一小格
- 按住时连续移动
- 回正应直接回到中间位置

### 曲库操作

- `Q`：将上轨当前曲目移到筛选库
- `Shift+Q`：将下轨当前曲目移到筛选库
- `E`：将上轨当前曲目移到精选库
- `Shift+E`：将下轨当前曲目移到精选库

### 删除

- `F`：删除上轨当前曲目
- `Shift+F`：删除下轨当前曲目

删除行为要求：

- 若当前曲目来自普通库：直接移到回收站
- 若当前曲目当前就位于回收站：弹彻底删除确认
- 删除完成后，对应轨道立即清空
- 删除逻辑需与主播放器现有行为保持一致

### 百分比跳转

- `·`：上轨跳到 `0%`
- `1` ~ `0`：上轨跳到 `10%` ~ `100%`
- `Shift+·`：下轨跳到 `0%`
- `Shift+1` ~ `Shift+0`：下轨跳到 `10%` ~ `100%`

说明：

- 这里的 `·` 指数字 `1` 左侧那个键
- 代码层通常对应 `Backquote`
- 实现时需兼顾不同键盘布局的键值差异

## 明确不提供快捷键的操作

以下操作当前阶段不提供快捷键：

- `Beat Sync`
- `MASTER`
- 装载歌曲到上轨 / 下轨
- 设置 bar line
- 网格左移 / 右移
- `Esc` 兜底取消

## 实现注意事项

### 1. 键盘事件优先级

- `CUE` 必须使用 `keydown + keyup` 成对处理
- `crossfader` 长按连续移动也不能只靠 click 语义
- 输入框聚焦、弹窗打开、确认框显示时，应禁用双轨快捷键

### 2. Shift 的职责必须单一

本草案里，`Shift` 的主要职责是“切换到下轨”。
只有 crossfader 这一组例外：`Shift+W/S/Up/Down` 统一解释为“回正”。
实现时不要再给 `Shift` 叠加第三层含义，否则规则会迅速变乱。

### 3. 删除后的轨道状态

删除上轨或下轨当前曲目后：

- 要同步清空对应 deck 的当前歌曲
- 要同步重置该 deck 的播放状态
- 不应让 UI 仍残留旧曲目信息

### 4. 百分比跳转键

由于不同键盘布局对 `·`、反引号键、`~` 的显示不同，实现时应优先按物理位置或兼容常见别名处理，避免热键只在部分键盘上生效。

## 验收要点

- `Space` 与 `Shift+Space` 只分别控制上轨 / 下轨，不串轨
- `C` 与 `Shift+C` 的行为与按钮完全一致，支持按住预听、松手返回
- `A/D` 与方向键左右只能按小节跳转，不能再表现为按秒快进快退
- `W/S` 与方向键上下可连续推动 crossfader
- `Shift+W/S/Up/Down` 必须稳定回正
- `Q/E/F` 与其 `Shift` 版本都只作用于对应轨道
- 普通库删除直接进回收站，不弹确认
- 回收站内删除才弹彻底删除确认
- 删除后对应轨道必须清空
