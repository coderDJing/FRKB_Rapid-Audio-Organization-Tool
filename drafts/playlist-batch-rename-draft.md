# 歌单批量重命名曲目文件名草案

本文档用于记录“从歌单右键进入，按模板批量重命名曲目文件名”功能的已确认需求。

## 范围

- 入口位于歌单右键菜单，和“批量补元数据”“转码”放一组。
- 只支持筛选库、精选库。
- 右键普通歌单时，仅处理该歌单。
- 右键歌单文件夹时，递归处理该文件夹下全部歌单。
- 不支持从 mixtape 歌单或 mixtape 歌单文件夹进入。
- 只改物理文件主名，不改扩展名，不移动目录。
- 不回写音频元数据标签。

## 模板与变量

- 使用自由模板。
- 变量只能通过点击标签插入。
- 用户手打的 `{title}` 这类内容一律当普通文本，不解析成变量。
- 允许手打普通固定文本、空格、连接符。
- 变量以可视化标签块展示，和普通文本区分。
- 变量标签不分组，全部平铺展示。
- 变量标签区域自动换行。
- 点变量标签时，插入到当前光标位置；如有选中文本，则替换选中文本。
- 模板输入框只做长度软提示，不做硬限制。

### 已确认变量

- `{title}`
- `{artist}`
- `{bpm}`
- `{key}`
- `{album}`
- `{genre}`
- `{label}`
- `{year}`
- `{trackNo}`
- `{fileName}`
- `{albumArtist}`
- `{discNo}`
- `{comment}`
- `{duration}`

### 变量渲染规则

- 空值占位词基于软件当前界面语言。
- `zh-CN` 使用 `未知`。
- `en-US` 使用 `unknown`。
- 数字类变量空值也统一使用占位词。
- 文本类变量先规范化后再参与渲染：
  - 换行、Tab 转空格
  - 连续空格压成一个
  - 去除首尾空格
- `{key}` 按当前界面调性显示方式渲染。
- `{bpm}` 按当前界面 BPM 显示方式渲染。
- `{duration}` 使用 `03m45s` 形式。
- `{trackNo}` 和 `{discNo}` 固定两位数，如 `01`、`02`。
- `{year}` 能识别成日期就只取年份，识别不了就原样输出。
- `{fileName}` 取原文件主名，不带扩展名。
- 模板中的普通文本不做智能整理；除了字段值规范化，其他按模板原样输出。

## 模板预设

- 支持保存、重命名、删除、设为默认。
- 删除预设前需要二次确认。
- 预设全局通用。
- 首版内置一套默认模板：`{title} - {artist}`。
- 内置默认模板允许直接覆盖。
- 若删除的是当前默认预设，则自动把内置默认模板重新设为默认。
- 每次打开弹窗时，默认载入“上次最后使用的预设”。
- 当前预设被修改后，点击保存时弹出：
  - 覆盖当前预设
  - 另存为新预设
- 预设名不允许重名。
- 预设列表顺序：
  - 默认预设置顶
  - 其他按创建时间排序
- 切换预设时，如果当前有未保存草稿，弹出：
  - 保存
  - 不保存
  - 取消
- 未保存草稿可以直接预览和执行。
- 关闭弹窗后，未保存草稿直接丢弃。

## 预览与示例

- 模板编辑区提供实时示例和完整预览两套反馈。
- 实时示例显示 3 首。
- 实时示例优先挑有代表性的曲目，尽量覆盖正常、占位词、冲突/异常等情况。
- 完整预览通过“生成预览”按钮手动触发。
- 模板、预设、语言、调性显示样式、BPM 显示格式变化时：
  - 实时示例即时刷新
  - 旧完整预览直接清空
- 完整预览不分页，使用虚拟列表滚动。
- 当右键的是歌单文件夹时，生成完整预览前显示扫描进度。
- 递归处理多个歌单时，完整预览打平展示，不显示来源歌单列。
- 预览列表支持按原文件名 / 新文件名模糊搜索。
- 预览搜索框占位文案：`搜索原文件名或新文件名`
- 预览筛选支持：
  - 全部
  - 异常
  - 已勾选
  - 未勾选
- 预览顶部批量勾选操作支持：
  - 全选
  - 全不选
  - 仅选正常项
  - 反选
- “仅选正常项”只选真正会执行的 `可执行` 项，不包含 `无需修改`。
- 搜索或筛选后，批量勾选操作只作用于当前结果集。
- 预览默认列：
  - 勾选
  - 原文件名
  - 新文件名
  - 状态
- 预览排序：
  - 异常项置顶
  - 其余按原始扫描顺序
- 正常项默认勾选。
- 异常项默认不勾选。
- 预览页不允许手动改单首生成结果，只能回去改模板。
- 预览列表中的文件名列使用单行省略号，hover 查看完整内容。
- 模板为空时不允许生成预览，并提示：`模板不能为空`

## 校验与状态

- 非法字符不自动替换、不自动删除。
- 文件名过长 / 路径过长不自动截断。
- Windows 与 macOS 都要按各自平台规则校验非法文件名。
- Windows 保留文件名（如 `CON`、`PRN`、`AUX`、`NUL`、`COM1` 等）视为非法文件名。
- 预览状态使用细分原因展示。
- 新文件名与原文件名完全一致时，仍显示在预览中，但执行时自动跳过。
- 异常项允许用户取消勾选后继续执行其他正常项。
- 像非法字符、超长这类硬错误，只要用户取消勾选，也允许继续执行剩余正常项。

## 冲突与重命名规则

- 同名冲突自动补序号。
- 序号格式为：
  - `文件名 (1)`
  - `文件名 (2)`
- 预览中直接显示最终会落地的文件名，包含自动补好的序号。
- 序号分配按原始扫描顺序。
- 若存在链式改名，执行时使用临时文件名做两阶段处理，保证可完成：
  - 例如 `A -> B`
  - `B -> C`
- 仅大小写不同也算有效改名，正常执行。

## 执行

- 执行前必须先生成完整预览并确认。
- 批量执行采用串行逐首处理。
- 执行前对勾选项再校验一遍。
- 若执行前发现文件失效、缺失、被占用或状态变化：
  - 该项记失败或跳过
  - 其余继续
- 执行过程中支持取消。
- 取消后未开始项停止，已完成项不回滚。
- 若单首执行失败：
  - 记失败
  - 继续执行后续项
- 如果当前正在播放的曲目在执行范围内，允许执行，完成后播放器和列表路径自动更新。
- 若用户在执行中关闭弹窗，弹出：
  - 继续后台执行
  - 取消执行
  - 返回
- 选择“继续后台执行”后：
  - 使用底部全局进度条
  - 跑完后自动弹结果弹窗

## 执行结果

- 结果弹窗显示：
  - 汇总数字
  - 失败列表
  - 跳过列表
- 被用户手动取消勾选的项，计入“跳过”，并明确标注“用户取消勾选”。
- 结果弹窗列表列为：`原文件名` / `目标文件名` / `状态`
- 结果弹窗汇总数字为：`总数` / `成功` / `失败` / `跳过`
- 执行前不额外显示风险提示。
- 中途取消执行后，结果弹窗仍然弹出。
- 取消后尚未开始的曲目：
  - 在结果列表中显示为“已取消”
  - 不计入汇总里的“跳过”
- 只有本次真的发生取消时，结果弹窗才额外显示“已取消”分组。
- 结果弹窗中的 `失败` / `跳过` / `已取消` 列表按原始扫描顺序展示。

## 已确认文案

- 右键菜单项：`批量重命名`
- 弹窗标题：`批量重命名`
- 生成预览按钮：`生成预览`
- 执行按钮：`开始重命名`
- 保存预设按钮：`保存预设`
- 设为默认按钮：`设为默认`
- 删除预设按钮：`删除预设`
- 重命名预设按钮：`重命名预设`
- 未保存草稿状态文案：`未保存`
- 结果弹窗操作按钮：仅 `关闭`
- 保存分支弹窗主选项：`覆盖当前预设` / `另存为新预设`
- 执行中关闭弹窗选项：`继续后台执行` / `取消执行` / `返回`
- 预览正常项状态文案：`可执行`
- 预览“文件名未变化”状态文案：`无需修改`
- 预览“非法字符”状态文案：`包含非法字符`
- 预览“文件名或路径过长”状态文案：`文件名或路径过长`
- 预览“用户取消勾选”状态文案：`手动跳过`
- 预览“文件不存在/失效”状态文案：`文件不存在`
- 执行时“文件被占用”状态文案：`文件被占用`
- 执行时“权限问题”状态文案：`权限不足`
- 执行时通用失败状态文案：`执行失败`
- 自动补序号后的正常项不单独区分，仍显示：`可执行`
- 取消后未开始项状态文案：`已取消`
- 预览搜索/筛选结果为空时文案：`没有匹配结果`

## 当前已知实现约束

- 改名后需要同步更新应用内依赖该物理路径的引用，包括但不限于：
  - 当前歌单列表显示
  - 其他引用该物理文件的普通歌单
  - 播放器当前播放状态
  - 分析队列路径映射
  - 其他基于文件路径的缓存或引用

## 收口结论

- 除上述已明确确认项外，剩余交互细节默认按本草案中的建议值落地。
- 当前草案已可作为实现输入使用。
